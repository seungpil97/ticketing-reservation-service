## 기술 회고

- Q: ddl-auto를 validate로 전환한 이유가 무엇인가요?
- A: create나 update로 두면 JPA가 엔티티 기준으로 DDL을 자동 실행해 운영 환경에서 의도치 않은 스키마 변경이 발생할 수
  있다. validate로 전환하면 Flyway가 만든 스키마와 엔티티가 일치하는지만 검증하고 직접 DDL은 실행하지 않는다.

- Q: Flyway migration 파일을 수정하지 않는 이유는?
- A: Flyway는 파일명과 내용의 체크섬으로 이미 적용된 파일을 추적한다. 기존 파일을 수정하면 checksum mismatch로
  애플리케이션 구동이 실패한다. 변경이 필요하면 항상 새 버전 파일을 추가해야 한다.

- Q: @PageableDefault로 기본값을 설정한 이유는?
- A: 클라이언트가 파라미터를 생략했을 때 서버 기본값(page=0, size=20, sort=id,desc)이 일관되게 적용되도록 하기
  위해. DB에서 ORDER BY로 정렬하므로 Java에서 전체 조회 후 정렬하는 방식보다 효율적이다.
