- dev/test DB 분리: 테스트 실행 시 개발 데이터가 오염되는 것을 방지하기 위해
- HOLD 상태 도입: 좌석 선점과 예약 확정을 분리해 일정 시간 내 결제하지 않으면 자동 해제되는 티켓팅 흐름을 구현하기 위해
