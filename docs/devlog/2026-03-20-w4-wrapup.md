## 기술 결정
- 비관적 락 → Redis 분산락 전환: 비관적 락은 DB 커넥션을 점유해 대용량 트래픽 시 DB 부하가 집중되는 문제가 있음. Redis 분산락은 DB 외부에서 락을 관리하므로 DB 부하 분산 가능
- 락 키 설계 `hold:seat:{showtimeId}:{seatId}`: 같은 회차의 같은 좌석에 대한 요청만 직렬화하고 다른 좌석 요청은 서로 영향을 주지 않도록 설계

## 트러블슈팅
- Spring Boot 4.x + Redisson 호환성 문제
  - `ClassNotFoundException: RedisAutoConfiguration` 발생
  - Spring Boot 3.4.3으로 다운그레이드하여 해결
  - Spring Boot 4.x는 아직 실무 도입 사례가 적고 주요 라이브러리 호환성이 불안정
- 중복 인덱스 경고
  - V9 migration 적용 시 기존 수동 생성 인덱스와 중복 발생
  - DROP INDEX 후 재생성으로 해결
